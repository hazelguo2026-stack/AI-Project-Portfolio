# 1. 产品概述

## 1.1 产品愿景

智枢笔记致力于成为知识工作者的"第二大脑"——一款兼具极致美学、强大功能与流畅体验的 Web 端笔记应用。产品核心理念是：**让知识的记录、整理与再创造变得令人愉悦，而非负担。**

在信息爆炸的时代，研究员、工程师、作家、产品经理等专业人士每天需要处理大量非结构化信息。现有工具要么功能简陋（如 Apple Notes），要么学习成本极高（如 Obsidian），要么 AI 能力不足（如 Notion）。智枢笔记的目标是在优雅与强大之间找到最佳平衡点。

## 1.2 产品定位

| 维度 | 定位描述 |
|------|----------|
| 用户群体 | 专业知识工作者，25–45 岁，高学历，对效率工具有深度需求 |
| 核心价值主张 | AI 原生的高颜值知识管理工具，让知识流动起来 |
| 竞争差异化 | 三位一体：编辑器体验 (Bear 级别) + 知识组织 (Notion 级别) + AI 能力 (原生深度集成) |
| 商业模式 | Freemium：免费版基础功能，Pro 订阅解锁 AI 功能与高级特性 |

## 1.3 产品目标

**短期目标（0–6 个月，MVP 阶段）：**
- 上线核心编辑器与文件夹组织功能
- 达到 5,000 注册用户
- 日活跃用户（DAU）/ 注册用户比 ≥ 30%

**中期目标（6–18 个月，v1.0 阶段）：**
- 完整 AI 功能上线（写作助手、智能搜索、自动标签）
- 付费转化率 ≥ 8%
- 用户 NPS ≥ 50

**长期目标（18 个月+）：**
- 成为专业知识工作者首选的 Web 端笔记工具
- 月活用户 50 万+
- 建立 AI 知识图谱能力，支持跨笔记知识关联

## 3.2 富文本 + Markdown 编辑器

### 3.2.1 功能概述

编辑器是产品的核心交互界面，目标是提供"写作心流"体验——用户专注于内容本身，工具完全服从内容意图，而非与之对抗。

### 3.2.2 详细功能需求

**[P0] 基础编辑能力**

- 支持标准富文本格式：加粗、斜体、下划线、删除线、行内代码
- 支持六级标题（H1–H6），一键格式切换
- 支持有序列表、无序列表、任务清单（TODO List，支持勾选交互）
- 支持引用块（Blockquote）
- 支持水平分割线
- 支持文字颜色与背景高亮颜色（调色板提供 16+ 种预设色）
- 支持超链接插入与编辑（链接预览卡片）
- 支持撤销/重做，历史记录深度 ≥ 100 步
- 支持全局查找与替换（支持正则表达式）

**验收标准：**
- 所有基础格式操作响应时间 < 16ms（不掉帧）
- 格式快捷键与 Notion/Bear 保持一致，降低学习成本
- 10 万字以上长文档编辑流畅度评分 ≥ 4/5（用户测试）

---

**[P0] Markdown 语法支持**

- 实时解析 Markdown 语法，边输入边渲染（"魔法 Markdown"模式）：
  - `**文字**` → 加粗
  - `# ` 开头 → 自动转为 H1
  - `- ` / `* ` → 无序列表
  - `1. ` → 有序列表
  - `- [ ] ` → 任务清单
  - ` ``` ` → 代码块
  - `> ` → 引用块
  - `---` → 分割线
- 支持在所见即所得（WYSIWYG）与纯 Markdown 源码视图之间一键切换
- 支持 Markdown 格式导出（`.md` 文件）
- 支持从 Markdown 文件导入（`.md` / `.txt`）

**验收标准：**
- Markdown 语法触发渲染延迟 < 50ms
- 所有 CommonMark 规范中的标准语法均支持
- 源码视图与渲染视图内容完全一致，无信息丢失

**[P0] 代码块**

### 3.3.2 详细功能需求

**[P0] 文件夹基础操作**

- 文件夹操作：创建、重命名、移动、删除（含二次确认）、复制
- 支持文件夹图标自定义（从预设 Emoji / 图标库选择，≥ 200 种图标）
- 支持文件夹颜色标记（8 种预设颜色）

- 支持文件夹下笔记计数显示
- 支持"所有笔记"视图（不区分文件夹的全量视图）
- 支持"最近编辑"智能文件夹（自动收录最近 7 天修改的笔记）

**验收标准：**

## 3.4 AI 功能

### 3.4.1 AI 写作助手

**功能概述：** 原生集成于编辑器内的 AI 助手，用户可在写作过程中随时调用，无需切换至外部工具。

**[P0] 基础 AI 写作操作**

触发方式：
1. 选中文本后弹出浮动工具栏，包含 AI 操作按钮
2. 在段落中输入 `/ai` 唤起 AI 命令面板
3. 侧边栏 AI 面板（持久化输入区域）
4. 快捷键：`Ctrl+Shift+A` / `Cmd+Shift+A`

核心 AI 操作：
- **续写（Continue Writing）：** 基于当前内容上下文继续生成后续内容
- **改写（Rewrite）：** 保留语义，用不同表达方式重写选中内容
- **润色（Polish）：** 优化表达，提升文字流畅度与可读性，修正语法错误
- **扩写（Expand）：** 将简短内容扩展为更详细的段落
- **缩写/摘要（Summarize）：** 提炼核心观点，生成精简版本
- **翻译（Translate）：** 支持中英文互译，及其他主要语言

AI 结果呈现方式：
- Streaming 流式输出，逐词实时展示生成过程
- 生成完成后提供：接受（Accept）/ 重新生成（Retry）/ 放弃（Discard）三个操作
- 支持"接受并继续"（接受结果后继续在结果末尾生成）

**验收标准：**
- AI 首 Token 响应时间（TTFT）< 2 秒（P95）
- 生成速度 ≥ 50 tokens/秒（流式展示）
- 用户操作到 AI 面板展示延迟 < 300ms
- AI 生成内容格式与当前笔记格式保持一致（标题/列表/代码块风格匹配）

---

**[P1] 对话式 AI 助手**

- 侧边栏 AI 对话面板，支持多轮对话上下文
- 自动注入当前笔记内容作为对话上下文（可开关）
- 支持针对特定笔记提问（"帮我总结这篇笔记的三个核心观点"）
- 支持跨笔记提问（"比较这两篇笔记的主要区别"）
- 对话历史保存，支持命名与回溯
- AI 回复中的代码/列表可一键插入到当前笔记

**验收标准：**
- 多轮对话上下文保持连贯（≥ 10 轮对话历史）
- 跨笔记查询响应时间 < 5 秒
- 对话历史持久化，App 重启后不丢失

---

**[P1] 自定义 AI 提示词（Prompt）**

- 支持用户保存自定义 AI 指令（"Prompt 模板"）
- 内置提示词模板库（≥ 20 个常用模板，按职业分类：研究员/工程师/作家/产品经理）
- 支持在提示词中插入变量（如 `{{selected_text}}`、`{{note_title}}`）
- 用户自定义提示词列表，可增删改排序

**验收标准：**
- 自定义提示词调用与内置功能体验一致
- 变量替换准确率 100%
- 提示词模板库用户满意度 ≥ 4/5

### 3.4.2 智能搜索 / 语义检索

**[P0] 基础全文搜索**

- 全局搜索快捷键：`Ctrl+K` / `Cmd+K`（Command Palette 模式）
- 支持笔记标题、正文内容的全文索引搜索
- 搜索结果实时展示（边输入边出结果）
- 搜索结果高亮匹配词
- 支持搜索范围过滤：当前文件夹 / 全部笔记 / 指定标签

**验收标准：**
- 10,000 篇笔记的全文搜索响应时间 < 500ms
- 搜索结果准确率（Precision@5）≥ 90%
- 搜索框展开动画 < 200ms

---

**[P1] 语义搜索**

- 基于向量嵌入（Vector Embedding）的语义检索，理解自然语言查询意图
- 示例：搜索"机器学习优化方法"能找到包含"梯度下降"、"Adam 优化器"的笔记
- 语义搜索与关键词搜索结果融合展示，按相关性排序
- 搜索结果卡片显示匹配原因（"该笔记讨论了 XXX，与您的查询相关"）
- 支持自然语言提问式搜索（"帮我找所有关于 React 性能优化的笔记"）

**向量化策略：**
- 笔记保存/修改后异步触发向量化（不阻塞用户操作）
- 向量存储：使用 pgvector 或专用向量数据库（如 Pinecone / Weaviate）
- 嵌入模型：优先使用 OpenAI text-embedding-3-small（成本与效果平衡）

**验收标准：**
- 语义搜索响应时间 < 2 秒（P95）
- 语义搜索相关性评分（NDCG@5）≥ 0.75
- 新笔记向量化完成时间（后台）< 30 秒

---

**[P1] 搜索过滤器**

- 日期范围过滤：今天 / 本周 / 本月 / 自定义范围
- 标签过滤（多选）
- 文件夹过滤
- 内容类型过滤：含代码块 / 含图片 / 含公式 / 含任务清单
- 支持过滤条件组合（AND 逻辑）

**验收标准：**
- 过滤条件变更后结果刷新 < 300ms
- 过滤状态持久化在当前搜索会话中

### 3.4.3 AI 自动标签 / 分类

**[P1] 自动标签建议**

- 笔记保存后（延迟约 3 秒），AI 分析内容并推荐 3–5 个标签
- 推荐逻辑：
  - 优先从用户已有标签库中匹配（保持标签体系一致性）
  - 其次推荐新标签（当内容涉及新主题时）
- 标签建议以非侵入式 UI 展示（底部浅色提示条，可一键接受全部 / 逐个选择 / 忽略）
- 用户接受/拒绝行为参与模型微调（Feedback Loop）

**验收标准：**
- 标签推荐准确率（用户接受率）≥ 60%
- 建议展示不阻塞编辑，延迟展示（保存后 3–5 秒）
- 用户可在设置中关闭自动标签功能

---

**[P2] 自动归档建议**

- AI 分析笔记内容，推荐最合适的文件夹（"这篇笔记建议归入：技术 / 前端开发"）
- 归档建议仅作推荐，用户确认后执行（非自动移动）
- 支持"不再对此类笔记提示"快速偏好设置

**验收标准：**
- 归档建议准确率（用户接受率）≥ 50%
- 建议提示可一键忽略
- 推荐逻辑可解释（"因为该笔记包含 React 代码和前端相关内容"）

| 全文搜索响应 | < 500ms（10K 笔记） | 前端计时 |
| 语义搜索响应 | < 2 秒（P95） | 后端 API 响应时间 |
| AI 首 Token 延迟 | < 2 秒（P95） | 流式请求计时 |

## 4.2 可靠性与数据安全

**数据持久化：**
- 自动保存：用户停止输入后 1 秒触发保存（防抖机制）
- 本地草稿缓存：在网络断开时写入 IndexedDB，恢复网络后同步
- 服务端数据备份：每日全量备份，保留 30 天历史版本
- 版本历史（Version History）：支持查看笔记最近 50 个历史版本（Pro 功能）

**数据安全：**
- 传输加密：全程 HTTPS（TLS 1.3）
- 静态加密：数据库字段加密存储（AES-256）
- 用户数据隔离：严格多租户隔离，禁止跨用户数据访问
- 端对端加密（E2E）：可选功能，Pro 用户可开启（开启后 AI 功能受限）
- 合规：符合 GDPR 及《个人信息保护法》（中国）

**账号安全：**
- 支持邮箱 + 密码、Google OAuth、GitHub OAuth、微信扫码登录
- 支持双因素认证（2FA，基于 TOTP）
- 登录异常（新设备/新地区）邮件提醒
- 会话管理：支持查看并远程注销设备

## 4.3 可用性与可访问性

**浏览器兼容性：**
- Chrome ≥ 100、Firefox ≥ 100、Safari ≥ 15、Edge ≥ 100
- 移动浏览器：iOS Safari ≥ 15、Android Chrome ≥ 100

**响应式设计：**
- 全面支持三种视口：桌面（≥ 1280px）、平板（768–1279px）、手机（< 768px）
- 手机端提供"简化写作模式"（聚焦编辑器，隐藏侧边栏）

**可访问性（Accessibility）：**
- WCAG 2.1 AA 级别合规
- 全键盘可操作（所有功能可通过键盘完成）
- 屏幕阅读器兼容（ARIA 标签完整）
- 色彩对比度：正文 ≥ 4.5:1，UI 元素 ≥ 3:1

**离线能力：**
- PWA 支持：应用可安装至桌面
- 离线模式：已加载的笔记可在无网络时编辑
- 离线编辑内容在重新联网后自动同步，冲突提示用户手动解决

## 4.4 扩展性需求

- 注册用户规模：支持百万级用户（水平扩展架构）
- 单用户笔记数量：支持 ≥ 10,000 篇笔记
